## 0. 데이터 품질 — sentinel 9999 (중요)
- **cape 83%·cinn 97%가 9999 sentinel**(= 대류불안정 없음/미계산). **tcoh는 전부 0(상수)**.
- 마스킹 전 cape↔solar +0.25는 **9999(맑은 안정일)가 만든 허위상관**이었다. 9999→NaN 처리 후 실값만 분석.
- → cape/cinn은 실값 비율 17%/3%로 표본 작음. **tcoh 제거.** (※ 다른 forecast 변수도 9999류 sentinel 주의)

## 1. 상관(마스킹 후) — 모델 잔차 기준이 후처리 실효성의 핵심
resid = 실측−예측 (음수 = 우리가 과대예측). 약함 기준 |r|<0.25.
- solar: gust resid_spear −0.22, hpbl −0.15, tcog −0.14, cape(실값) −0.18 — 모두 약~중간.
- wind: gust↔util +0.85(강하나 wind_spd와 중복 → resid ~0), tcog resid +0.22, hpbl +0.16 — 약~중간.
- 단일 변수 선형상관으론 강한 후처리 신호 없음.

## 3. 결론 — tcog만 의미 있고 해석도 명확
`tcog`(대류운/graupel 총량)는 표준예보가 못 잡는 **대류성 날씨**를 플래그한다:
- **대류일(tcog>0) → solar 과대예측(−0.069)**: 대류운이 일사를 더 깎는데 예보 일사엔 덜 반영.
- **같은 날 wind 과소예측(+0.132)**: 대류성 gust가 지속풍속보다 강한데 못 잡음.
- solar는 깎고 wind는 올리는 방향이 **둘 다 물리적으로 일관** = 사용자가 가설한 forecast 전용 신호.
- 단 드묾(낮 표본 ~7%) → 전체가 아닌 **대류일 표적 보정**.

나머지(사용자 기준 "큰 차이·해석 없으면 미사용"):
- **cape·cinn**: sentinel 지배·실값 약함 → 미사용. **tcoh**: 상수0 → 미사용.
- **gust**: wind_spd와 중복(잔차 ~0). solar에선 −0.045로 보조 가능성만. **hpbl**: wind 중간(혼합고).

## 4. 후속(후처리 단계, 서빙 본체 다음)
- **tcog 후처리 1개**만 가볍게: tcog>0이면 solar util ↓·wind util ↑ (그룹 잔차로 적합).
- **평가는 랜덤 스플릿(계절 골고루)**: forecast 데이터가 6개월 단일 시즌이라 시간순 분할은 편향(사용자 지시).
- gust/hpbl은 tcog와 함께 다변량으로 소폭 보강 여부만 점검(단독 채택 안 함).
