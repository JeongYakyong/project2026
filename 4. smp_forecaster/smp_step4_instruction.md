# smp_step4_instruction.md — 4단계 SMP 다듬기: 잔차 분해 + Conditional Quantile

> **이 문서를 받은 Claude Code(CC)에게**
> 4단계(SMP)를 *새 모델로 갈아엎는 작업이 아니다*. 검증된 A안(DA 가격선 + 음수경보)을 **유지·확장**하고,
> "왜 RT는 예측 불가인가"를 데이터로 **입증**하는 게 목적이다.
> §0(분석 전제)은 그대로 리포트에 옮겨 쓸 근거이므로 정확히 지킬 것. §1의 "하지 말 것"을 어기지 말 것.
> 무거운 작업 전에 §3 끝의 **Decision Gate**를 반드시 통과시키고, 추측으로 값·단위 채우지 말 것.

## 0. 분석 전제 — 왜 이렇게 모델링하는가 (★리포트용 근거★)

이 섹션은 기획서 §2(제안배경)·§3(차별성)에 그대로 쓰인다. 모델 설계 철학의 토대.

### 0.1 DA SMP는 "임의 가격"이 아니라 결정론적 최적화의 산물
한국은 CBP(Cost Based Pool, 변동비반영시장) 구조다. 발전사가 가격을 부르는 게 아니라,
KPX가 **하루 전 오후 3시**에 다음을 계산해 가격결정 발전계획을 세운다:

1. 익일 **시간대별 수요를 예측** (기상·계절·요일 반영)
2. 발전기를 **변동비 싼 순서로 급전**(원자력 → 석탄 → LNG → 유류)
3. 예측 수요를 채우는 **마지막(가장 비싼) 발전기의 변동비 = 그 시간의 SMP**

→ 즉 **DA SMP ≈ f(수요예측, 연료비, 급전순위)** 의 결정론적 함수다.
우리 3단계 출력(net_load)과 기상으로 상당 부분 재현 가능 → **DA를 가격선 앵커로 채택하는 근거.**

### 0.2 RT SMP는 비관측 시장행동이 지배 → magnitude 예측 불가 (주가와 동형)
RT는 "하루 전 계획"이 아니라 **실제 그 시각의 실시간 수급 불균형**이 반영된다. DA 대비 추가되는 것:
- **수요예측 오차**: KPX 익일 예측 vs 실제 수요의 차이
- **신재생 실시간 변동**: 제주는 태양광·풍력 비중이 높아, 한낮 태양광 과잉 시 net_load 급락 → SMP 급락/음수
- **시장참여자 실시간 의사결정**: 발전기 입찰행동, 계통운영자 재급전, 예비력 조정 — **외부에서 관측 불가**

→ 이 비관측 부분 때문에 RT magnitude는 **주가처럼 점예측 불가**.
**실측 증거(trial_error §3): 수요·신재생을 완벽 실측으로 입력해도 RT 시간별 MAE 16.3이 천장.
shape 오라클(11.3)과의 5점 격차 = 어떤 피처로도 설명 안 되는 시장 고유노이즈.**

### 0.3 우리의 입장 (= A안이 정당한 이유)
- 가격선(level)은 **DA가 무적** → 점예측으로 덮지 않고 DA를 그대로 앵커로 쓴다 (비음수 구간 MAE 8.3).
- RT 잔차 중 **유일하게 설명 가능한 부분 = "신재생 과잉 → 음수"** → 이것만 정량화한다.
- "경험 많은 사람도 날씨 보고 안다"는 반박에 대한 답:
  사람은 이 구조를 **정성적**으로 알 뿐. 우리는 **비관측 부분과 설명가능 부분을 데이터로 분리**해
  후자를 **확률·강도**로 정량화한다(사람이 못 주는 정보). ← 이게 차별성의 핵심 문장.

## 2. 이번 작업 목표 (3개)

| # | 작업 | 성격 |
|---|------|------|
| **P1** | RT−DA 잔차 분해 → "예측 불가 + 음수만 설명가능" **입증** | 분석·리포트 근거 |
| **P2** | 음수 발생 시 **깊이·지속의 conditional 분포** 예측 (A안 위 overlay) | 결과물 확장 |
| **P3** | A안 이진 경보 **유지** (안 건드림) | 검증된 자산 보존 |

순서 원칙: **P1을 먼저** 끝내 입증 그림을 확보 → Decision Gate → P2. P3은 손대지 않는다.

## 3. Phase 1 — RT−DA 잔차 분해 (예측 불가 입증)

**대상은 RT가 아니라 `잔차 = smp_jeju_rt − smp_jeju_da`.**
(RT 자체를 STL 돌리면 가격 레벨이 trend/seasonal에 흡수돼 "DA로 설명되는 부분"과 "진짜 노이즈"가 안 갈라진다. DA가 레벨을 잡고 남은 잔차를 분석하는 게 우리 3층 서사와 맞다.)

**해야 할 것:**
1. 학습창(2024-06~)에서 `잔차 = rt − da` 시계열 구성 (RT NULL 시점은 제외, 보간 금지).
2. STL 분해(period=24, 일 주기) → trend / seasonal / residual. **분해 그림 1장** 저장.
3. **분산 분해 표**: 잔차의 총분산 중 (a) seasonal로 설명, (b) 기상 피처 회귀로 설명, (c) 설명 안 되는 순수 노이즈 비율.
   - (b): 잔차(또는 STL residual)를 기상 피처(solar_util, solar_rad, wind, hour, month 등 A안 피처)로 회귀 → R² 보고.
   - (c) = 1 − (a) − (b). **이 (c)가 "예측 불가 시장노이즈"의 정량 증거.**
4. **음수만 설명가능 확인**: 잔차를 음수구간/비음수구간으로 나눠, 기상 R²가 음수구간에서 유의미하게 높음을 보일 것(= 우리가 음수만 정량화하는 정당성).

- [ ] "완벽 실측 입력으로도 RT MAE ≈ 16, shape 오라클 11"의 격차를 분산 분해와 연결한 **3~5줄 요약**

## ★ Decision Gate (Phase 1 → Phase 2) ★

P2 착수 전 아래를 **먼저 측정해 보고**하고, 그 결과로 P2 방식을 정한다. (임의 진행 금지)

1. **음수 샘플 수 세기**: 학습창(2024-06~)에서 `rt < 0` 인 시간 수 N을 보고.
   - trial_error 기준 음수 ~137건 → **샘플이 매우 적을 수 있음.**
   - **N ≥ 300**: LGBM quantile regression 진행 (P2-A).
   - **N < 300**: quantile 회귀는 과적합 위험 → **경험적 조건부 분포(empirical)** 로 대체 (P2-B). 무리한 회귀 금지.
2. 음수 사건의 **지속(연속 시간 run-length)** 분포와 `smp_rt_neg_num`(시간내 15분 0~4 심도)의 분포를 함께 보고 → 지속/심도 중 무엇이 데이터로 잡히는지 판단.

→ 이 게이트 결과(N, 분포)를 사용자에게 보고하고 **P2-A / P2-B 선택을 받은 뒤** 진행.

## 4. Phase 2 — 음수 깊이·지속 정량화 (A안 overlay 확장)

**대전제: 비음수 구간은 절대 안 건드린다.** 출력은 *경보가 켜진 구간에만* 얹는 conditional overlay다.
(전 구간 RT를 점예측하던 옛 실패를 반복하지 않기 위함.)

### P2-A. LGBM Conditional Quantile (게이트에서 N≥300일 때)
- **타깃**: `rt < 0` 조건부 깊이(magnitude). 음수 샘플에만 학습.
- **모델**: LGBM `objective='quantile'`, `alpha = 0.1 / 0.5 / 0.9` 3개 (P10/P50/P90).
- **피처**: A안 10피처 재사용(smp_jeju_da, net_load, nl_lead_1/2, solar_util, wind_util, wind_spd_west, solar_rad_south, hour, month). 새 피처 최소화.
- **검증**: 음수 샘플이 적으므로 시계열 split + 음수구간 pinball loss 보고. 과적합 징후 시 피처 축소.

### P2-B. 경험적 조건부 분포 (게이트에서 N<300일 때)
- (hour × solar 수준) 또는 (hour × month) 격자별 **음수 깊이의 경험적 P10/50/90** 룩업표 작성.
- 회귀 대신 분포 자체를 보여줌 — 적은 샘플에서 정직하고 과적합 없음.

### 지속(duration) — 부차 목표 (게이트 2번 결과에 따라)
- 옵션 a: 연속 음수 run-length의 경험적 분포(사건이 시작되면 평균 N시간) 보고.
- 옵션 b: `smp_rt_neg_num`(시간내 심도 0~4)을 순서형 심각도로 활용.
- 무리하면 생략 가능. **깊이(P2-A/B)가 must, 지속은 best-effort.**

### 최종 출력 형태 (데모/리포트에 보여줄 그림)
> "내일 12~14시 음수 위험 82%(이진 경보). 발생 시 예상 깊이 P50 −35 / P90 −68원, 예상 지속 ~3h."
