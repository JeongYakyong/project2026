## ★ 먼저 읽어야 할 핵심 발견 2가지 (결론에 직결)

1. **`제주_시간대별발전량(16-24).csv`에는 발전량 데이터가 없다.**
   파일명과 달리 내용은 **기상·파생변수**(기온·강수량·풍속·습도·일사·적설·운량 + Hour_sin/cos 등 엔지니어링 피처)뿐이다.
   연료원별은커녕 발전량 컬럼 자체가 없다. 기간도 2020-01-01 ~ 2022-10-23로 "16-24"와 불일치.
   → **도출식의 좌변 "발전량" 항을 이 파일에서 얻을 수 없다.**

2. **연료원별 분해는 `제주_시간대별전력거래량(17-25).csv`에 이미 들어 있다.**
   컬럼이 `LNG / Oil / Other / Solar / Wind`로, **시간별 연료원별 거래량(=발전량 proxy)**이다.
   즉 **LNG 발전량은 역산할 필요 없이 직접 컬럼으로 존재**한다(태양광·풍력도 별도 컬럼).
   → 원래 계획한 도출식(`전체수요 − HVDC − 신재생`)은 **불필요**해진다. (자세한 판정은 §6.3)

## 6.2 교차 분석 (§3)

**① 도출식 기간 교집합** — 원 도출식은 불필요(아래 6.3 Q1)하나, 형식상 교집합을 계산하면:
- 연료원별(전력거래량): 2017-01-01 ~ **2025-01-01**
- HVDC(마스터): 2017-01-01 01:00 ~ 2025-04-01
- (이름만) 발전량 파일: 2020-01-01 ~ 2022-10-23 — **발전량 없음이라 무의미**
- **실효 공통 윈도우(연료원별 LNG 직접 사용 기준): 2017-01-01 ~ 2025-01-01 (시간별)**. HVDC를 함께 쓰면 동일 구간 전부 커버됨.

**② 해상도 사다리(집계 필요 지점)**

| 데이터 | 원 해상도 | 기간 | 제주 LNG 연결 시 필요한 집계 |
|---|---|---|---|
| 제주 연료원별(LNG 등) | **시간별** | 2017-01 ~ 2025-01 | 기준 데이터 |
| 제주 HVDC | 시간별 | 2017-01 ~ 2025-04(+스냅샷 2026-03) | 동일 시간축 |
| KPX 화력발전량(제주 포함) | 일별 | 2018-01 ~ 2022-06 | 제주 LNG를 **일별 합계**로 집계해 대조 |
| daliy_lng_gen(전국) | 일별 | 2021-01 ~ 2026-03 | 제주를 일별 집계(단 전국이라 직접 분해는 불가, 상관분석용) |
| 기온효과(월별 시계열) | 월별 | 2007-01 ~ 2021-06 | 제주 LNG를 **월별 합계**로 집계 |
| 수입단가 | 월별 | 2010-01 ~ 2026-01 | 월별 집계 후 join |
| 요금단가(xlsx) | 월별 | 2020-01 ~ 2026-06 | 월별 집계 후 join |
| 유가 | 일별 | 2020-01 ~ 2026-05 | 일/월 집계 양쪽 가능 |
| 기온민감도(룩업표) | (시계열 아님) | — | 계수표로 별도 적용 |

→ **제주 시간별 → 일별 → 월별** 두 단계 집계만 하면 KOGAS 월/일 데이터와 모두 정합.

**③ 단위 정합** — 제주 `전력거래량`(LNG/Oil/Solar/Wind)과 `HVDC`는 **둘 다 MW(시간별, ≈MWh/h)**로 동일 → 더하고 빼는 연산 정합. KPX 화력은 일별 **MWh**(이미 적산값)이라 제주 시간합(MWh)과 동일 차원. KOGAS `공급량`은 물량(추정 톤)으로 **차원 다름** → 발전량과 직접 뺄셈 불가, 상관/스케일링만.

**④ HVDC 중복 위험** — 마스터 파일에서 `HVDC_Total = HVDC1+2+3` 가 전 구간 오차 0으로 검증됨. → **단일 출처 권고: `제주_시간대별HVDC(17-25).csv`의 `HVDC_Total`만 사용.** 스냅샷 9개는 마스터 종료(2025-04-01) 이후를 연장할 때만 보조로 사용(중복·결손 주의: 2025-07~09 결손).

## 6.3 ★최종 판정 (Decision Gate)★

**Q1. 제주 LNG(내연) 발전량을 도출식으로 계산 가능한가?**
→ **도출식 불필요 — LNG 발전량이 직접 제공됨(예).**
근거: `제주_시간대별발전량(16-24).csv`에는 발전량이 전혀 없어 원 도출식의 "발전량" 항을 그 파일에서 얻을 수 없다. 그러나 **추가 입수된 `ONLY_GEN.csv`가 시간별 `LNG_Gen`(및 `Oil_Gen`, `HVDC_Total`)을 직접 제공**하고, `제주_시간대별전력거래량(17-25).csv`도 연료원별(LNG·Oil·Solar·Wind)을 제공한다. 따라서 **LNG는 역산 없이 직접 컬럼으로 확보**된다. 도출식(`수요−HVDC−신재생`)은 불요.

**Q2. 가능하다면 어떤 시간 해상도·기간인가?**
→ **시간별로 가능.**
- **권장 1차 소스 `ONLY_GEN.csv`: 2020-01-01 00:00 ~ 2024-12-31 23:00** (중복 9월 제거 후 5년 정시 완전셋, MW≈MWh/h).
- 더 긴 구간이 필요하면 `제주_시간대별전력거래량`의 LNG 컬럼: **2017-01-01 ~ 2025-01-01**.
- 일별 검증은 KPX 화력발전량(제주도, MWh, 2018-01~2022-06). HVDC 연장이 필요하면 2025-04(마스터)·2026-03(스냅샷, 7~9월 결손)까지 가능.

**Q3. 제주 LNG 수요를 KOGAS 가스 데이터와 연결하려면 어느 단계에서 어떤 집계가 필요한가?**
→ 제주 시간별 LNG를 **(a) 일별 합계**, **(b) 월별 합계** 두 축으로 집계한다.
- **월별 합계** → 요금단가(xlsx, 2020-01~2026-06, 원/Nm³·원/GJ), 수입단가($/MMBTU, 2010-01~2026-01), 기온효과(2007-01~2021-06) 와 `연월` 키로 join.
- **일별 합계** → daliy_lng_gen(전국, 2021-01~2026-03) 및 KPX 화력(제주, 2018-01~2022-06) 과 대조.
- 단, `daliy_lng_gen`·요금단가는 **전국 단위**이고 제주 LNG는 지역 단위이므로 직접 분해가 아니라 **상관/탄력성 분석**으로 연결해야 함. 단위도 발전량(MWh) vs 가스물량(톤)·단가(원,$)로 달라 정규화/스케일링 필요.
- 기온민감도(`montly_lng_gen_temp`)는 시계열이 아닌 **월별·기온편차별 지수표**이므로, 시계열 join이 아니라 기온 변수에 대한 **계수 적용** 방식으로 사용.

### 부록 — 정리/전처리 권고(다음 단계 착수 시)
1. 제주 3개 시계열은 헤더 `일시`가 cp949로 저장되어 ISO-8859로 오디코드됨 → **`encoding='cp949'`로 통일 로드**, 컬럼 strip.
2. HVDC는 `HVDC_Total` 단일 사용. 스냅샷은 필요 시에만 wide→long melt 후 마스터 뒤에 append(중복/결손 점검).
3. 요금단가 7개 xlsx는 병합셀 un-pivot 표준화 → `연월 / 구분(일반발전·집단) / 단위 / 항목(원료비·공급비·합계) / 값` long 테이블로 통합.
4. KPX 화력 컬럼명 공백 strip, 제주도 필터로 제주 LNG 일별 검증셋 구성.
5. 단위 미표기 파일(`daliy_lng_gen` 공급량, 유가 통화)은 원 출처 메타로 단위 확정 후 사용.
