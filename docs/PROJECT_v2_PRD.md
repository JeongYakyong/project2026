## 1. 검증 목표 (Thesis)

> 본 프로젝트는 "예측 정확도 자랑"이 아니라 **하나의 명제를 데이터로 입증**하는 것이 목적이다.

- **검증목표 1 (제주)**: `est_net_load`로 **발전용 천연가스 수요량의 변화를 예측할 수 있다.**
  신재생(태양광·풍력)이 가장 많이 들어오는 제주에서, 신재생→잔여부하(net_load)→가스 발전수요의 연결을 입증.
- **검증목표 2 (전국)**: 제주에서 보인 관계가 **육지(전국)에서도 성립한다.**
  전국은 가스 발전량(`gen_gas_kr`)이 **실측으로 존재** → 데이터 질이 더 좋아 검증이 더 엄밀.

**한 줄**: 신재생 변동이 만든 잔여부하를 가스 발전이 메운다 — 그 관계를 제주에서 입증하고 전국 실측으로 확증한다.

## 5. 방법론 (확정)

### 5.0 데이터 단계 (모델링 선행 — 확보·분리) ★
> **이 단계가 끝나기 전에는 모델링(5.2~) 착수 금지.** (게이트 §9 G-6)

**(1) 확보 — 마스터셋 결합**
- 모든 소스를 `timestamp`(시간)/`date`(일)/`연월`(월) 키로 정렬·결합.
- 제주: `input_data_jeju.db` historical + `jeju_hvdc_hourly`(HVDC) + `only_gen`(실측 LNG/유류) + KOGAS(월) + 유가(일).
- 전국: `input_data_land.db` historical(net_load_kr·`gen_gas_kr` 등) + 동일 외생.
- 결합 직후 **결측·중복·시간 구멍을 먼저 집계 보고**(채우기 전에).
- LNG 타깃 시계열 구축은 §5.1 절차로 수행(도출+backfill).

**(2) 분리 — 정답 vs 피처 (누수 차단)**

| 구분 | 컬럼 | 규칙 |
|---|---|---|
| **검증 타깃(정답)** | LNG 발전량 (제주: only_gen 실측 / 도출분, 전국: `gen_gas_kr`) | **피처로 절대 사용 금지** |
| **모델 입력(피처)** | net_load(real/est), 달력(hour·dow·month·`day_type`), 기온(3지점), 계절·연도추세 | 정답·정답파생 미포함 |
| **금지 피처(누수원)** | HVDC, 유류 발전, 타깃 lag, RT SMP 등 발행지연 변수 | 타깃 도출/지연에 연루 → 제외 |

- **자기참조 차단**: 도출 LNG(2025~2026)는 net_load에서 나온 값 → 이 구간은 **정확도 산출에서 제외**(데모 표시용). 엄밀 정확도는 ①제주 2020–2024 실측, ②전국 `gen_gas_kr`에서만 낸다.
- **HVDC·유류 제외 근거**: 둘 다 타깃 도출식에 들어간 값 → 피처로 넣으면 정답 누수. (예측 경로엔 forecast에 HVDC가 없어 자연 배제와도 일치.)

**(3) 분할 — 시간순(랜덤 금지)**
- train/val/test를 **시간순**으로. 레짐 경계(시범사업 2024-06, LNG 증설)를 가로지르지 않게 하거나 플래그.
- 권장: train 2020–2023 / val 2024(실측 정답 마지막 해) / test = forecast 구간(2025-12~). 전국 동일 골격.

**(4) 산출(게이트)**: 분리·라벨링된 학습셋 + 검증셋 + 데이터 딕셔너리(피처/타깃/금지 표시). **이게 끝나야 5.2 착수.**

### 5.1 과거 LNG 타깃 시계열 구축 (도출 — "단순")
1. `net_load = 계통수요 − 신재생` (DB의 `real_net_load_jeju` 사용. 정의 확인 §9 G-1)
2. `fuel_gen = net_load − HVDC` (= 유류 + LNG, ~2026-04 계산 가능)
3. LNG/유류 분해: **2024년 LNG:유류 비율 기준**으로 `fuel_gen`을 분할 → LNG·유류 추정, **2026-04까지 backfill**.
   - 2020~2024는 `only_gen` 실측을 **그대로 정답**으로 사용(도출 불필요).
   - (선택 개선, 비필수) 단일 스칼라 비율 대신 **부하수준별 비율**(merit order 반영)로 분할하면 충실도↑. 단순본으로 시작.

### 5.2 예측 경로 (서빙 — net_load → LNG)
- forecast에 HVDC가 없으므로, 위 타깃으로 **`LNG_gen = f(net_load, 달력, 기온, 계절)`** 학습 → `est_net_load_jeju`에 적용해 **LNG 발전 예측** 산출.
- HVDC는 함수가 암묵 흡수(서빙 시 별도 HVDC 입력 불요).

### 5.3 가스 수요·비용 환산 (KOGAS 연결 = 자격 핵심)
- 예측 LNG 발전량(MWh) → **발전용 가스 수요/비용**으로 환산: `gas_tariff`(원/GJ·Nm³)·`gas_import_price`($/MMBTU)·`gas_temp_effect` join(월 해상도 집계).
- KOGAS 데이터는 **외부 기준점** — net_load 도출과 독립이라 자기참조 없는 sanity check 역할도 겸함.

### 5.4 검증 계층 (정직성 = 강점)
| 레벨 | 정답 | 자기참조 | 산출 |
|---|---|---|---|
| 제주 2020–2024 | only_gen 실측 LNG | 없음 | **MAE/MAPE (핵심 수치)** |
| 제주 2025–2026 | 도출 LNG | 있음(주의) | 데모 연속성, "추정" 표기 |
| **전국** | `gen_gas_kr` 실측 | **없음** | **검증목표 2 — 최강 증거** |
| KOGAS 연결 | 가스 공급량·단가 | 없음 | 방향성 정합 확인 |
