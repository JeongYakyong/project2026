# visual.md — 전국 신재생 발전 지도 시각화 지시서 (CC 핸드오프)

> **이 문서를 받은 Claude Code에게**: 아래 사양과 **레퍼런스 프로토타입(`renewable_capacity_map.html`)** 을 기준으로,
> Streamlit 앱에 들어갈 **① 신재생 강도 지도 + ② 하단 권역 테이블**을 구현해줘.
> 디자인은 프로토타입을 충실히 재현(임의 변경 금지). 데이터는 더미 → 실 forecast DB로 교체.
> **§1 Decision Gate를 먼저 통과**시키고 작업을 시작할 것(의사결정 전 무거운 작업 금지).

## 0. 배경 / 목적

발전사업자가 "**어느 권역의 신재생 발전이 오늘·앞으로 강한가**"를 한 화면에서 보게 하는 인포그래픽.
- PROJECT.md 매핑: **Track B(전국 확장 미리보기, §3.1 Tab 4)** + 기획서 **§4.4 확장 로드맵 차트** + **§4.5 스크린샷 패키지**.
- 핵심 메시지: 전국은 SMP 비대상이므로 **net_load(=신재생 발전 강도) 시각화**로 가치를 보여준다.
- 기존 `weather_map.py`(Plotly/IDW)가 있으나, **디자인 기준은 프로토타입 쪽이 채택됨**. 둘을 합치는 게 이 작업의 골자.

## 1. ★먼저 — 사용자 확인이 필요한 결정 (Decision Gate)★

CC가 임의로 정하지 말고, 정량 근거/선택지만 정리해 사용자 확인 받을 것.

1. **구현 방식 — Leaflet 임베드 vs Plotly 재구현**
   - **A안(Leaflet HTML 임베드)**: 프로토타입을 `st.components.v1.html`로 그대로 삽입. *디자인 100% 보존*, 단 Streamlit 위젯↔지도 상태 연동이 제한적(날짜·토글을 HTML 내부 JS로 처리하고, 실데이터는 렌더 시 주입).
   - **B안(Plotly 재구현)**: `weather_map.py`처럼 `Choroplethmapbox`(feature별 `fillOpacity`로 단색 강도) + `Scattermapbox`(이모지 텍스트) + `hovertemplate`(카드). *forecast DB·위젯 연동 쉬움*, 단 hover 카드/이모지 디자인을 Plotly로 재현하는 손이 듦.
   - 권고: **운영·실데이터 연동 우선이면 B안**, 디자인 충실도 절대 우선이면 A안. → 사용자 결정.
2. **하단 테이블 컬럼/정렬** (§5) — 제안값 그대로 갈지, 가감할지.
3. **제주 관측소 데이터** — 제주(고산 등) ASOS/예보를 **실제 수집**하는지. 보간 대체 금지(§7).
4. **활성도 bin 임계값** — 본 문서 값은 **잠정**. 사용자가 추후 일사·풍속 분포로 탐색 예정 → CC는 잠정값 + `# TODO: 사용자 탐색` 주석으로만 둘 것.
5. **일사 정규화 방식** — forecast의 `radiation_`(W/m²)을 "일사 비율(0~1)"로 바꿀 때, **청천일사(태양고도 기반) 정규화**로 갈지 **전운량 fallback**으로 갈지(§3.3).

## 2. 최종 디자인 사양 (프로토타입 기준)

레퍼런스: **`renewable_capacity_map.html`** (이 문서와 함께 전달). 아래는 그 동작의 명세.

### 2.1 지도(면)
- **단색(초록) choropleth.** 색 진하기 = **신재생 발전 강도**. 진할수록 강함, 연할수록 약함/설비 없음.
- 강도 = `Σ(설비용량 × 활성도)` = 상대 기대 발전량. (자세한 식 §3.4)
- **투명도 상한 40%** → `fillOpacity ≤ 0.60`. 약한 권역도 최소 `0.06`. 바탕 지도가 항상 비쳐야 함.
- 빨강/파랑 "우세 자원" 색칠은 **금지**(개표지도처럼 보임). 우세 정보는 hover 카드에서만.
- 시도 경계는 `skorea_provinces_simplified.json` 사용. 같은 권역의 시도는 같은 색 → 자연스럽게 권역으로 보임. 내부 경계선은 옅게(`#cbd5e1`, weight 0.7), hover 시 해당 도형 테두리 진하게.

### 2.2 권역 라벨(원 대체)
- 권역 중심에 **하늘상태 이모지 + 권역명** 라벨만. (원/버블 제거됨)
- 이모지 = 전운량+강수 기반(§3.3).

### 2.3 Hover 카드 (★100점, 그대로 재현★)
다음 항목을 검은 카드로:
```
[권역명]  D+n
구성 시도 (예: 대구·경북)
[이모지] 하늘상태 · 풍속 N.N m/s        기상: [관측소명]
🔴 태양광 용량   X,XXX MW
🔵 풍력 용량       XXX MW
☀️ 태양광 활성도   NN%  (등급)   [막대]
🌀 풍력 활성도     NN%  (등급)   [막대]
```

### 2.4 컨트롤
- **태양광/풍력 체크박스**: 강도 계산에 어느 발전원을 포함할지 필터(둘 다 기본 ON). 우세 비교용 아님.
- **날짜 칩**: D+1·D+2·…·D+7·(간격)·D+12. 칩에 하늘상태 이모지 + 신뢰도 점(초록 높음/주황 보통/빨강 낮음). D+8~D+12는 "참고(신뢰도 낮음)".
- **좌하단 요약**: 단일 초록 바(전국 평균 가동 강도) + 가장 활발한 권역 2곳 + 대표 하늘상태.

### 3.3 활성도 BIN 기준 (★잠정 — 사용자 탐색 예정★)
**태양광** (입력: 일사 비율 r = 실측/청천, 0~1; 강수 시 강등)
| 일사 비율 | 활성도 | 등급 |
|---|---|---|
| ≥0.80 | 95% | 매우 좋음 |
| 0.60~0.80 | 75% | 좋음 |
| 0.40~0.60 | 52% | 보통 |
| 0.20~0.40 | 30% | 낮음 |
| <0.20 / 강수 | 10% | 매우 낮음 |

**풍력** (입력: 풍속 m/s; 터빈 파워커브)
| 풍속(m/s) | 활성도 | 등급 |
|---|---|---|
| <3 | 0% | 정지(cut-in 미만) |
| 3~6 | 30% | 약함 |
| 6~9 | 65% | 양호 |
| 9~13 | 95% | 최적(정격) |
| 13~25 | 85% | 강풍 |
| >25 | 0% | 차단(cut-out) |

**하늘상태 이모지** (전운량 0~10, 기상청 하늘상태 기준 + 강수형태 우선)
- 강수: 🌧️ 비 / 🌨️ 눈 · 0~5: ☀️ 맑음 · 6~8: ⛅ 구름많음 · 9~10: ☁️ 흐림

> **forecast 컬럼 연결**: `radiation_*`(W/m²)→일사비율(정규화 §1.5), `wind_spd_10m_*`(m/s)→풍속, 전운량/하늘상태 컬럼→이모지. 단기예보에 일사 직접값이 없으면 **전운량으로 일사비율 근사**.

### 3.4 강도(=면 색) 식
```
권역 활성도:  sa = solar_bin(일사비율),  wa = wind_bin(풍속)
권역 강도(gen) = (태양광ON? 태양광용량 × sa : 0) + (풍력ON? 풍력용량 × wa : 0)
maxGen = max_권역( 활성용량 ) × 0.95           # 활성용량 = 켜진 발전원 용량 합
score = min(1, gen / maxGen)                   # 0~1
fillOpacity = 0.06 + 0.54 × score              # 상한 0.60
```
- 의미: **용량이 크고 + 그날 기상이 좋아야** 진해짐. 설비 없는 권역은 항상 옅음.
- 잠정 상수: `GREEN="#059669"`, `OP_MIN=0.06`, `OP_MAX=0.60` (코드 상단에 모아둘 것).

### 3.5 날짜/신뢰도
- D+1~D+7: 단기·중기예보. D+8~D+12: 중기예보 범위 끝/밖 → **신뢰도 낮음 표기**, 활성도는 참고값.
- D+11~D+12는 표준 예보 범위 밖 → 평년/앙상블 또는 참고용 더미. 칩 점 색으로 신뢰도 구분.
