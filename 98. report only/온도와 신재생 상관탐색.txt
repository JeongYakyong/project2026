[개요] 기온효과 대비 신재생예측 효과 비교 분석 데이터 구성 안

1. 추천 데이터 스키마 (일별 또는 시간별 권장)
- date : 분석 기준 일자 (YYYY-MM-DD)
- gas_power : 발전용 가스 공급량 (★신재생 효과 분석의 핵심 타깃)
- gas_civil : 도시가스 공급량 (기온/난방 효과 분석의 핵심 타깃)
- hdd (난방도일) : max(18°C - 평균기온, 0) -> 겨울철 난방 수요 선형화
- cdd (냉방도일) : max(평균기온 - 22°C, 0) -> 여름철 전력/냉방 수요 선형화
- renew_gen : 신재생 실제 발전량 (태양광+풍력, MWh) -> 가스 발전 대체 효과 측정
- renew_error : 신재생 예측 오차량 (|실제량 - 예측량|) -> 가스 피크 유발 효과 측정

2. 단순 상관관계(Correlation) 분석의 한계와 보완
- 비선형성 문제: 기온과 가스량은 U자형 관계이므로 raw 기온 대신 HDD, CDD 변환 필수
- 다중공선성 문제: 여름철 폭염 시 기온(CDD)과 태양광(renew_gen)이 동시에 증가하므로, 두 변수가 서로 얽혀 단순 상관계수만으로는 독립적 영향력 분리 불가

3. 추천 분석 프로세스
- 1단계 (기초 분석): HDD, CDD 변환 후 각 변수 간 피어슨 상관계수 및 방향성 확인
- 2단계 (상대적 기여도 비교): 가스 공급량을 종속변수(Y)로 두는 다중회귀모형 구축
  * 산식: Y(발전용가스) = b0 + b1*HDD + b2*CDD + b3*renew_gen + b4*renew_error
  * 팁: 각 변수의 단위가 다르므로 반드시 '표준화(Z-score)' 후 회귀계수(Beta) 절대값 크기로 영향력 순위 비교
- 3단계 (고도화): XGBoost / LightGBM 등 머신러닝 모델 학습 후 변수 중요도(Feature Importance) 및 SHAP 값 산출하여 시각화

4. 분석 핵심 포인트 (인사이트)
- 용도별 분리: 도시가스는 기온(HDD)의 독무대이므로, 신재생 효과 비교는 반드시 '발전용 가스'를 타깃으로 진행해야 함
- 신재생 변수 이원화:
  * 신재생 '발전량' 증가 -> 가스 공급량 감소 (부의 상관관계)
  * 신재생 '예측오차' 증가 -> 가스 공급량 변동성/피크 증가 (정의 상관관계)
